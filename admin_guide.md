## 2. Установка и запуск локально

### 2.1. Клонирование репозитория

```bash
git clone https://github.com/JayceWright/CoffeeStockAI.git
cd CoffeeStockAI
```

### 2.2. Создание виртуального окружения

```bash
python -m venv venv

# Windows:
venv\Scripts\activate

# Linux/macOS:
source venv/bin/activate
```

### 2.3. Установка зависимостей

```bash
pip install -r requirements.txt
```

Список ключевых зависимостей ([requirements.txt](file:///c:/Antigravity/CoffeeStockAI/requirements.txt)):

| Пакет | Назначение |
|---|---|
| `fastapi` | Веб-фреймворк API |
| `uvicorn[standard]` | ASGI-сервер |
| `sqlalchemy>=2.0` | ORM для работы с БД |
| `pandas` | Обработка датасетов |
| `prophet` | ML-модель временных рядов |
| `matplotlib` | Построение графиков |
| `slowapi` | Rate Limiting (защита API) |
| `openai>=1.0` | Клиент OpenRouter (AI-объяснения) |
| `python-dotenv` | Загрузка переменных окружения |

### 2.4. Настройка переменных окружения

Создайте файл [.env](file:///c:/Antigravity/CoffeeStockAI/.env) в корне проекта (пример — раздел 6).

### 2.5. Запуск сервера

```bash
uvicorn app.main:app --reload --port 8000
```

После запуска система доступна по адресу: **http://localhost:8000**

> **Примечание:** При первом запуске система автоматически инициализирует базу данных ([coffeestock.db](file:///c:/Antigravity/CoffeeStockAI/coffeestock.db) для SQLite или подключается к PostgreSQL через `DATABASE_URL`) и заполняет таблицу [ingredients](file:///c:/Antigravity/CoffeeStockAI/app/main.py#75-99) начальными данными (seed).

## 4. Управление базой данных

### 4.1. Модели данных

Система использует 5 ORM-моделей SQLAlchemy:

| Таблица | Назначение |
|---|---|
| [ingredients](file:///c:/Antigravity/CoffeeStockAI/app/main.py#75-99) | Справочник ингредиентов (запасы, поставщики, минимальные уровни) |
| [order_drafts](file:///c:/Antigravity/CoffeeStockAI/app/main.py#304-407) | Черновики заказов, генерируемые ML-моделью |
| `feedback_log` | Лог корректировок менеджера (дельты для bias correction) |
| `sales_history` | История транзакций из POS-терминалов |
| `forecasts` | Сохранённые прогнозы Prophet |

### 4.2. Локальная СУБД (SQLite)

При отсутствии переменной `DATABASE_URL` система автоматически использует SQLite:
```
coffeestock.db  ← создаётся автоматически при первом запуске
```

### 4.3. Продакшн СУБД (PostgreSQL на Railway)

На Railway используется managed PostgreSQL 16. Строка подключения задаётся автоматически через переменную `DATABASE_URL`.

### 4.4. Сброс базы данных

Чтобы сбросить все данные и начать заново (только локально!):
```bash
# Удалить SQLite-файл:
del coffeestock.db

# Перезапустить сервер — база создастся заново:
uvicorn app.main:app --reload --port 8000
```

## 5. Обучение и запуск ML-модели

ML-пайплайн запускается отдельно от веб-сервера как CLI-скрипт.

### 5.1. Запуск прогнозирования

```bash
python ml/forecast.py
```

### 5.2. Что делает скрипт

1. **Загрузка данных** — читает [Data/Coffee Shop Sales.xlsx](file:///c:/Antigravity/CoffeeStockAI/Data/Coffee%20Shop%20Sales.xlsx) (149 116 транзакций) и [Data/open-meteo-40.74N74.04W7m.csv](file:///c:/Antigravity/CoffeeStockAI/Data/open-meteo-40.74N74.04W7m.csv) (погода NYC, 181 день)
2. **ETL** — агрегация по [(Дата, Локация, Тип продукта)](file:///c:/Antigravity/CoffeeStockAI/static/index.html#454-462), формирование временных рядов
3. **Обучение Prophet** — для каждого `product_type × store_location` строится отдельная модель с параметрами:
   - `weekly_seasonality=True`
   - `changepoint_prior_scale=0.3`
   - Регрессоры: `temperature`, `rain`, `is_hot`, `is_rainy`
4. **BOM-разузлование** — перевод прогнозов напитков в расход ингредиентов
5. **Валидация WMAPE** — метрика рассчитывается на последних 10% данных по 6 ключевым ингредиентам
6. **Визуализация** — сохраняет [ml/forecast_plot.png](file:///c:/Antigravity/CoffeeStockAI/ml/forecast_plot.png)
7. **Сводка** — выводит таблицу факт/прогноз по всем ингредиентам на 30 дней

### 5.3. Ожидаемый вывод

```
✅ Обучение ансамбля завершено (Prophet + CatBoost)!

🎯 Валидация на уровне ингредиентов (test=10% дат): WMAPE = 10.9%
   Цель по KPI 3.1: MAPE < 15% — ✅ Достигнуто!
💣 Взрываем BOM...

📋 СВОДКА (по ингредиентам):
  cup_pcs              | факт (30д):     1412 | прогноз:     1551
  milk_ml              | факт (30д):    49241 | прогноз:    52880
  coffee_beans_g       | факт (30д):    11955 | прогноз:    13255
  ...
```

## 6. Переменные окружения (.env)

Создайте файл [.env](file:///c:/Antigravity/CoffeeStockAI/.env) в корне проекта со следующими переменными:

```env
# База данных (если не задана — используется SQLite)
DATABASE_URL=postgresql://user:password@host:5432/coffeestock

# OpenRouter API (для AI-объяснений аномалий)
OPENROUTER_API_KEY=sk-or-v1-ваш_ключ_здесь

# Опционально: URL сайта для OpenRouter заголовков
YOUR_SITE_URL=https://your-app.railway.app
YOUR_SITE_NAME=CoffeeStockAI
```

> **Важно:** Файл [.env](file:///c:/Antigravity/CoffeeStockAI/.env) добавлен в [.gitignore](file:///c:/Antigravity/CoffeeStockAI/.gitignore) и **никогда не должен попадать в репозиторий**.

### 6.1. Получение ключа OpenRouter

1. Зарегистрируйтесь на [openrouter.ai](https://openrouter.ai)
2. Перейдите в раздел **Keys** → **Create Key**
3. Скопируйте ключ в [.env](file:///c:/Antigravity/CoffeeStockAI/.env) → `OPENROUTER_API_KEY`

## 7. Деплой на Railway

### 7.1. Первичный деплой

1. Создайте аккаунт на [railway.app](https://railway.app)
2. Нажмите **New Project** → **Deploy from GitHub repo**
3. Выберите репозиторий `CoffeeStockAI`
4. Railway автоматически определит [Procfile](file:///c:/Antigravity/CoffeeStockAI/Procfile) и запустит:
   ```
   web: uvicorn app.main:app --host 0.0.0.0 --port $PORT
   ```

### 7.2. Добавление PostgreSQL

1. В проекте Railway нажмите **+ Add Service** → **Database** → **PostgreSQL**
2. Railway автоматически добавит `DATABASE_URL` в переменные окружения

### 7.3. Переменные окружения на Railway

1. Откройте сервис → вкладка **Variables**
2. Добавьте `OPENROUTER_API_KEY` вручную

### 7.4. CI/CD

Любой `git push` в ветку `main` автоматически запускает:
1. Сборку нового Docker-образа
2. Установку зависимостей из [requirements.txt](file:///c:/Antigravity/CoffeeStockAI/requirements.txt)
3. Перезапуск сервиса (zero-downtime deployment)

## 8. API-справочник

Интерактивная документация Swagger UI доступна по адресу: **http://localhost:8000/docs**

### Ключевые эндпоинты

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/` | Главная страница (перенаправление на [/static/index.html](file:///c:/Antigravity/CoffeeStockAI/static/index.html)) |
| `GET` | `/api/v1/orders/drafts` | Получить список черновиков заказов с прогнозами |
| `POST` | `/api/v1/orders/approve` | Утвердить заказы с корректировками менеджера |
| `POST` | `/api/v1/orders/send` | Отправить утверждённые заказы поставщикам |
| `GET` | `/api/v1/forecasts/summary` | Сводка прогнозов по ингредиентам |
| `POST` | `/api/v1/ai/explain` | AI-объяснение аномалии (OpenRouter) |
| `GET` | `/api/v1/feedback/stats` | Статистика цикла обратной связи |
| `POST` | `/api/v1/pos/sync` | Синхронизация данных POS (Rate Limited: 4 rps) |
| `POST` | `/api/v1/delivery` | Принятие поставки (обновление остатков) |

### Rate Limiting

Эндпоинт `/api/v1/pos/sync` защищён ограничением частоты запросов:
- **Лимит:** 4 запроса в секунду с одного IP-адреса
- **При превышении:** HTTP `429 Too Many Requests`

## 10. Устранение неисправностей

### ❌ Ошибка при запуске: `ModuleNotFoundError: slowapi`

```bash
pip install slowapi
```

### ❌ Ошибка: `CommandExecutionError` при Prophet

Prophet требует `cmdstanpy`. Установите Stan:
```bash
pip install cmdstanpy
python -c "import cmdstanpy; cmdstanpy.install_cmdstan()"
```

### ❌ 500 Internal Server Error при `/api/v1/orders/drafts`

Проверьте логи сервера. Скорее всего, ML-модель не запускалась и таблица [order_drafts](file:///c:/Antigravity/CoffeeStockAI/app/main.py#304-407) пуста. Запустите:
```bash
python ml/forecast.py
```

### ❌ AI-объяснения не работают (возвращают ошибку)

1. Проверьте наличие `OPENROUTER_API_KEY` в [.env](file:///c:/Antigravity/CoffeeStockAI/.env)
2. Убедитесь, что ключ действителен: https://openrouter.ai/keys

### ❌ PostgreSQL не подключается на Railway

1. Убедитесь, что сервис PostgreSQL добавлен в проект Railway
2. Проверьте, что `DATABASE_URL` автоматически добавлена в переменные окружения сервиса
3. Сделайте `git push` для перезапуска сервиса

### ℹ️ Логи сервера в Railway

1. Откройте проект в Railway
2. Перейдите в сервис → вкладка **Deployments**
3. Нажмите на последний деплой → **View Logs**
